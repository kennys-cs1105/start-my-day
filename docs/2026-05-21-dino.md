# DINO 系列

## DINO V3 在工业领域

核心：少样本 + 异常检测 + dense defect

优势：传统 CNN 的最大问题是过度依赖 `supervised label`，而 DINOV3 天然学习 `feature geometry`，对 `anomaly` 极度敏感

模型需求：few-shot；self-supervised；dense feature；local geometry；universal feature

## 典型 pipeline

经典流程

```
image
 → DINOv3 backbone
 → dense patch feature
 → memory bank
 → anomaly score
 → segmentation map
```

## PatchCore 风格

流行 DINOV3 -> PatchCore，因为 DINOV3 patch embedding 极强


## DINOV3 训练流程

1. 总体结构

DINOV3 本质：`student-teacher SSL`

结构：

```
image
    - global crops
    - local crops
    - masked crops
```

2. 整体伪代码

```
for images in loader:

    # multi-view augmentation
    global_views = ...
    local_views = ...

    # student
    s_out = student(all_views)

    # teacher
    with torch.no_grad():
        t_out = teacher(global_views)

    # dino loss
    loss_dino = dino_loss(s_out, t_out)

    # ibot loss
    loss_ibot = ibot_loss(...)

    # gram anchoring
    loss_gram = gram_loss(...)

    # total
    loss = (
        loss_dino
        + λ1 * loss_ibot
        + λ2 * loss_gram
    )

    loss.backward()
    optimizer.step()

    # ema update
    update_teacher()
```

## 工业级训练难点

1. batch size

DINO 类方法极大依赖于大batch，因为需要 `feature diversity`，否则容易出现 `collapse`

2. teacher temperature

非常敏感，通常设置为 `teacher_temp = 0.04`. 太低容易出现 `collapse`，太高容易出现 `semantic` 弱

3. center momentum

`center momentum = 0.9`，不稳定会直接炸

4. Gram loss 开启时机

不能太早，因为前期 `feature` 未稳定，通常在后 30%-50% 训练阶段开启
